RIFT Security Algorithms

   Registration Procedure(s)

 Specification Required

   Expert(s)

 Unassigned

   Reference
   [RFC9692]

   Available Formats
   [IMG]
   CSV

             Name                                                    Recommendation                                                Reference
                            Simplest way to ensure integrity of transmissions across adjacencies when used as outer keys and
   HMAC-SHA256              integrity of TIEs when used as inner keys. Recommended for most interoperable security           [SHA-2][RFC2104]
                            protection.
   HMAC-SHA512              Same as HMAC-SHA256 with stronger protection.                                                    [SHA-2][RFC2104]
                            Recommended for high security applications where private keys are protected by according nodes.
   SHA256-RSASSA-PKCS1-v1_5 Recommended as well in case not only integrity but origin validation is necessary for TIEs.      [RFC8017, Section 8.2]
                            Recommended when adjacencies must be protected without disclosing the secrets on both sides of
                            the adjacency.
   SHA512-RSASSA-PKCS1-v1_5 Same as SHA256-RSASSA-PKCS1-v1_5 with stronger protection.                                       [RFC8017]
